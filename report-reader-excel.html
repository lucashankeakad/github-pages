🎯 Estratégias de Processamento

📄 Processamento de Arquivos CSV ▼

O sistema oferece três abordagens distintas para processar arquivos CSV, cada uma otimizada para diferentes cenários.

CSV Bulk (CsvHelper + Lotes)
Utiliza a biblioteca CsvHelper para ler o arquivo e agrupa as linhas em lotes de 2000 registros. Imagine que você está lendo um livro e, a cada 20 páginas, passa essas páginas para um grupo de amigos resumir. É uma abordagem equilibrada que funciona bem para a maioria dos casos, oferecendo boa performance sem complexidade excessiva.
CSV Pipeline (System.IO.Pipelines)
A mais sofisticada, utilizando System.IO.Pipelines para processar o arquivo byte por byte. É como ter uma linha de montagem onde uma pessoa lê letra por letra e vai passando para outras formarem palavras e frases. Essa abordagem oferece o máximo de performance e eficiência de memória, mas requer conhecimento avançado de programação de baixo nível.
CSV Sequential (Linha por Linha)
A mais conservadora, processando uma linha por vez. É como ler um livro palavra por palavra, entendendo completamente cada frase antes de passar para a próxima. Ideal quando a memória é extremamente limitada ou quando você precisa de controle total sobre cada linha processada.

📊 Processamento de Arquivos Excel ▼

Para arquivos Excel, o sistema oferece cinco estratégias diferentes, cada uma com suas características únicas.

Sequential (ClosedXML Simples)
A mais simples, usando ClosedXML para carregar o arquivo e processar linha por linha. É a abordagem mais direta e fácil de entender, perfeita para arquivos pequenos ou quando a simplicidade do código é prioritária.
Parallel (ClosedXML + Multi-core)
Também usa ClosedXML, mas processa todas as linhas simultaneamente usando todos os cores do processador. É como distribuir páginas de um livro para várias pessoas lerem ao mesmo tempo. Oferece excelente performance para arquivos médios, mas consome mais memória.
Bulk (OpenXML + Lotes Paralelos)
Utiliza OpenXML (a biblioteca nativa da Microsoft) para ler o arquivo em grandes lotes de 5000 linhas, processando cada lote em paralelo. É um meio-termo entre performance e uso de memória, ideal para arquivos grandes em ambientes de produção.
Streaming (OpenXML + Channels)
A mais avançada, combinando OpenXML com Channels do .NET para criar um pipeline produtor-consumidor. Um thread lê o arquivo continuamente enquanto múltiplos threads processam os dados conforme chegam. É como uma esteira rolante onde o conteúdo flui continuamente do leitor para os processadores, ideal para arquivos gigantescos.
ToCsv (Conversão + Processamento)
Estratégia única: primeiro converte o Excel para CSV em disco, depois processa o CSV resultante. É como traduzir um livro técnico para linguagem simples antes de analisar. Embora adicione uma etapa extra, pode ser mais eficiente para certos tipos de dados e permite debug mais fácil.
